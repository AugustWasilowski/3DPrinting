In “MECH-anic” the “controls language” tells the user how to manipulate and activate the controls to trigger different behaviors. 

Dictionary
  1. Control - any object that can be manipulated by the user holding some state. A control can have its own behaviors and trigger criteria.
  2. State - the properties of an object at any one time. Examples ( on / off / 100% / 1,2 or 3 / active / non-active)
  3. Principle control (PC) - a control the user is attempting to manipulate.
  4. Trigger Criteria (TC) - other controls that must be triggered into a specific state following defined rules before the principle control can be triggered.
  5. Physical Control Mechanics (PCM) - the physical manipulation required to trigger a principle control once the “Trigger Criteria” have been fulfilled.
  6. Station - Any specified area holding a set of controls and identified to the user as a discrete area. 
  7. Trigger Group (TG) - the visual area holding “trigger criteria”. Rounded rectangular area.
  8. Indicator - shape / letter / number used to identify something in the game.
  9. Principle Control Indicator (PCI) - indicator for principle control.
  10. Control Indicator (CI) - visual identifier placed next to the physical location of the control along with any trigger criteria. Used in the trigger criteria to specify its required state.
  11. Sequence Arrow (SA) - Can indicate the order controls must be triggered in trigger criteria. Can connect Control Indicators or Synchronization Bars.
  12. Synchronization Bar (SB) - Indicates controls must be triggered together at the same time.
  13. Control Coordinates (CC) - when controls are organized in a grid each row and column will have an indicator. The combination of two indicators will specify the control.
  14. Flow - when the flow of any flow type must be shared between more than one control. i.e. the flow of oil or power through all controls.
  15. Flow Types (FT) - substance or entity that a control needs to be triggered. i.e. power / oil / ammo / water / electric eels / oxygen 
  16. Level - objects with states that are 0 to 100%. i.e. Power Level or Fuel level.
  17. Sub-Criteria (SC) - vertically arranged next to a Control Indicator, specifies the required state and rules to trigger the control properly to fulfill the criteria.
  18. Control Grid - when controls are arranged in rows and columns. Each row and column will have its own Control Indicator (CI).
  19. Control Coordinates (CC) - Controls in a Control Grid can be identified by two Control Indicators (CI) one for the row and one for column. Two CI’s together specify the control to trigger, these are the control’s Control Coordinates (CC).

Example: Trigger Group (TG) with Principle Control Indicator (PCI), with Trigger Criteria (TC) using a Synchronization Bar (SB)
Below we see a single solid thicker bar it is a Synchronization Bar (SB) meaning the Club and Heart control must be activated at the same time. Once activated at the same time and remaining active the Principle Control (PC) can be triggered. Note Synchronization Bars (SB) can be oriented horizontally, vertically or diagonally.

 

Example: Trigger Group (TG) with Principle Control Indicator (PCI), with Trigger Criteria (TC) using Synchronization Bars (SB)
Below we see Synchronization Bars (SB) connecting Control Indicators (CI). In this example order does not matter, the Club or Spade can be triggered first as long so the “synchronized” control is triggered with it. 
Potential viable sequences are:
  Club and Diamond together then Spade and Heart together.     
                                   OR
  Spade and Heart together then Club and Diamond together.

Example: Trigger Group (TG) with Principle Control Indicator (PCI), with Trigger Criteria (TC) using Synchronization Bars (SB) and a Sequence Arrow (SA).
This example shows the controls now do have a required order. The Sequence Arrow (SA) specifies which “synchronized” controls must go first, Club and Diamond, then Spade and Heart. Demonstrating the Sequence Arrow (SA) can be used to connect Control Indicators (CI) or Synchronization Bars (SB).
